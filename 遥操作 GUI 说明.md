# 遥操作 GUI 说明

目前需要和通信对接的有三个功能：**位置姿态指令控制、手柄控制、视觉识别回传与执行对准**

![image.png](https://alidocs.oss-cn-zhangjiakou.aliyuncs.com/res/1GXn45KKMx4ZKqDQ/img/98fcec4a-38ff-4f6f-b7ed-15ebfc89d964.png)

![image.png](https://alidocs.oss-cn-zhangjiakou.aliyuncs.com/res/1GXn45KKMx4ZKqDQ/img/20002803-0135-49f2-9e99-25d2f747e183.png)

*   左上角：**位置姿态指令控制：**输入期望的末端执行器位置（XYZ）与姿态（欧拉角，RPY）指令，机械臂执行
    
    *   以机械臂基座为坐标系原点（绝对）
        
    *   执行可选择两种算法，笛卡尔直线路径求解/逆运动学求解，对应两个按键
        
    *   这个功能更多的是**调试机械臂用**，和遥操作的应用场景没有直接关联，实际设计GUI的时候**占比尽量缩小**
        
    
*   左下角：**手柄控制**
    
    *   软件手柄控制为后续的硬件手柄驱动打基础，同时也算一种遥操作备用方案
        
    *   以末端相机为坐标系原点（相对），与右边的视觉结合，可以想象你在操作一架无人机
        
    *   左右半边：模拟摇杆，左边控制位置（X/Y，上下左右平移），右边控制视角（Yaw/Pitch，左右摆动和上下俯仰）
        
        *   注：Qt没有摇杆组件，这里的摇杆是后续重写的组件，设计时放两个Widget在这里就可以了，固定一下长宽比（比如200\*200）
            
    *   中上：激活勾选框，勾选时才会激活手柄控制，防止误触
        
    *   中下：模拟扳机（LT/RT），控制Z轴（伸入伸出）
        
    
*   左半边底部：指令控制与手柄控制状态栏（label），反馈控制状态（比如提醒将要发生碰撞、靠近奇异点等等）
    
    *   （内容后续补充）
        
    
*   右半边：**视觉识别回传**
    
    *   实时反馈经过视觉识别后的图像（直接画在上面）
        
    *   之前的视觉算法都已经实现了，这里只需要呈现，放一个长宽比640\*480的label即可
        
    
*   右半边底部：视觉识别状态反馈与执行对准按键（此处尚未明确具体实现目标，需要讨论）
    
    *   期望是能够在发现螺母目标后，一键对准，过程自动化，便于进行后续操作
        
    *   左边是label，反馈实时识别状态（比如，在……位置识别到螺母，是否进行对准？）
        
    *   右边按键，确认执行对准
        
    *   这里涉及到一个问题：如果画面中存在多个识别到的目标，怎么选择？
        
        *   初步设想：模拟硬件手柄的十字按键或XYAB键，用于切换锁定目标
